## Leakage And Injection Boundary

Checks performed:

- Inference configs do not point to `points_train_label.csv`, training labels, hidden answers, or score files.
- C03 and TSViT submission configs use only `/input`, `/output/result.json`, model checkpoints, normalization stats, and preprocessing constants.
- The local smoke used labels only to select a few known point IDs and query dates for a platform-shaped CSV. Those labels are not passed into `run_inference`.
- Result writing collapses duplicate JSON keys deterministically and reports duplicate conflicts; this prevents accidental conflicting duplicate-key injection.
- Platform C03 run reported `12` duplicate output-key rows but `0` conflicts.

Known boundary:

- Training and preprocessing scripts still contain label-handling code because they are used for model training. This is expected and should remain separated from inference configs.
- Do not tune from hidden-test feedback unless the mechanism is explainable and verified by local diagnostics. A single hidden score is not enough to justify arbitrary changes for the paper.
